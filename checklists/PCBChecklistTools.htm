Usage: use appropriate checklists. Checked state can be exported as ID-list (see buttons below) and re-imported from ID-list. ID lists are great to save checklist state with design files.
Some functions require you to press ctrl or shift while clicking for safety reasons.

CL0: Altium CAD Checklist

  1. Schematic:
    1. Verify Project Options (or proper project template)
      1. Check Project Options > Error reporting
      2. Check Project Options > Connection Matrix
      3. Check Project Options > Options > Net Identifier Scope
    2. Run ERC check by compiling the project: Project > Compile (cc)
    3. Check connectivity with alt+click, alt+dblclick on net or using net highlight colors (alt+click) (alt+doubleclick)
  2. PCB:
    1. Check Component links: Project > Component links... (ck)
    2. Check for latest Schematic version: Design > Import changes from... (di)
    3. Check for latest library files (tools > update from PCB libraries) but apply only if no local changes were made. Otherwise, update manually. (tl)
    4. DRC
      1. Check DRC settings and enabled DRC checks (dr), (td)
      2. Polygons and planes are set to direct connect with vias (design rules), Polygons and planes are set to thermal relief connect (typically four connections) with component pads
      3. In the Polygon Manager, check every Polygon's minimum neck width to be enabled and ≥ the minimum track width (tgm). Otherwise be aware that smaller necks are not reported by DRC as width or unrouted violations.
      4. Run DRC (tdr). Process every error.
    5. Mechanical layers are used correctly. The default is
      | M01: Board outline and routed cut-outs | M02: PCB Info
      | M11: Top Dimensions | M12: Bottom Dimensions
      | M13: Top Component Outline And 3D Body | M14: Bottom Component Outline And 3D Body
      | M15: Top Courtyard | M16: Bottom Courtyard
  3. Output generation:
    1. Use newest / applicable Output Jobfile

CL2: KiCad Checklist

  1. Schematic:
    1. Verify Project Options (or proper project template)
      1. Schematic Setup > Electrical Rules > Violation severity
      2. Schematic Setup > Electrical Rules > Pin Conflicts Map
      3. Schematic Setup > Project > Netclasses
    2. Run ERC check: Inspect > Electrical Rules Checker
    3. Check connectivity with highlight net or using net class colors ~ ^ # Ctrl+Shift+H
    4. Check for latest library files (Tools > update Symbols from Libraries...) but apply only if no important local changes were made. Otherwise, update manually.
  2. PCB:
    1. Check for latest Schematic version: Tools > Update PCB from Schematic... F8
    2. Check for latest library files (Tools > update Footprints from Libraries...) but apply only if no important local changes were made. Otherwise, update manually.
    3. DRC
      1. Check DRC settings and enabled DRC checks (Files > Board Setup... > Design Rules > ...) Ctrl+Shift+B
      2. Run DRC (Inspect > Design Rules checker) Ctrl+Shift+R. Process every error.
    4. Mechanical layers are used correctly.
  3. Output generation:
    1. Use newest / applicable Output Jobfile

CL1: Eagle CAD Checklist

  1. Schematic:
    1. Check connectivity with the show (eye) tool.
  2. PCB:
    1. Check PCB is in sync with Schematic (forward/back annotation enabled, active)
    2. Perform ratsnest, check that all polygons are pourred and no airwires remain
    3. DRC
      1. Check DRC settings (td)
      2. Run DRC (tdr)
    4. Check that only vector fonts are used
  3. Output:
    1. Use newest / applicable CAM Processor settings

ADPROC: Altium Procedures

  1. Back-Annotation (PCB RefDes → Sch):
    1. Sch: Project > Compile(cc)
    2. PCB: Check links: Project > Component links (ck), press Ok even, if no changes are reported.
    3. Verify board and schematic are up to date
      1. PCB: No changes reported for Design > Import changes from ... (di)
      2. PCB: No changes reported for Design > Update schematics in ... (du)
      3. Sch: No changes reported for Design > Update PCB Document... (du)
    4. PCB: Tools > Re-Annotate (tn)
    5. Sch: Project > Compile (cc)
    6. Sch: Project > Re-Compile (cr)
    7. PCB: Design > Import Changes from... (di)
    8. Save PCB and Sch
